## 📌 0. 코어 아키텍처 원장 준수 사항 (영구 지침)
- **5대 관심사 분리(SoC):** `AssetSummaryCard`, `AIRecommendationSection`, `AssetDetailTab`, `TransactionHistoryTab`, `TransactionDialog` 5분할 구조를 절대 유지하며, 최근 `React.lazy`를 통한 동적 임포트 전환 구조를 보존합니다.
- **성능 방어 결계:** 연산 부하가 큰 합산 데이터는 무조건 `useMemo` 내부에서 처리하여 렌더링 오버헤드를 0%로 통제합니다.
- **무결성 강제:** 백엔드 송출 직전 `quantity`, `price`, `tax_fee` 데이터는 반드시 `Math.abs()`로 감싸 양수 상태를 보장합니다.
- **도미노 Refetch:** 거래가 저장/수정되면 공통 API 모듈(`fe/src/api/index.js`)을 사용하여 데이터를 안정적으로 갱신합니다.

## 📌 1단계. UI/UX 전역 컬러 시스템 교체 (KRX 규격)
- **내용:** 기존 글로벌 파이낸스 컬러 룰을 파쇄하고, 한국 주식 시장 표준(KRX) 컬러 시스템으로 전면 교체합니다.
- **상세 요구사항:**
  - **상승 / 매수 / 평가이익 / 자산 증가:** 빨간색 계열(`var(--red-600)` 등) 적용
  - **하락 / 매도 / 평가손실 / 자산 감소:** 파란색 계열(`var(--blue-600)` 등) 적용
  - 최근 리팩토링된 CSS 클래스(예: `text-buy`, `text-sell` [cite: 20260520_refactor_summary.md])의 정의를 수정하여 대시보드 전역에 KRX 규격이 일괄 반영되도록 조치하십시오.

## 📌 2단계. 거래 내역 검색 UI 구현 (`TransactionHistoryTab`)
- **내용:** 대규모 매매 내역 속에서 원하는 조건의 데이터를 기민하게 추적할 수 있도록 검색 바를 신설합니다.
- **상세 요구사항:**
  - `TransactionHistoryTab` 레이아웃 상단에 PrimeReact 컴포넌트를 활용한 폼 영역을 배치하십시오.
    - **계좌 필터:** `Dropdown` 컴포넌트 연동
    - **종목 필터:** `InputText` 또는 `AutoComplete` 검색 기능
    - **기간 필터:** `Calendar` 컴포넌트의 `selectionMode="range"` 속성 적용
  - 사용자가 검색 조건을 변경할 때마다 공통 API 모듈(`fe/src/api/index.js`)의 `get` 메서드를 통해 백엔드 확장 API(`GET /api/transactions`)를 호출하도록 이벤트를 바인딩하십시오.

## 📌 3단계. 계좌별 보유 종목 목록 필터링 및 투명도 처리 (`AssetDetailTab`)
- **내용:** 자산 청산 종목을 포함한 전체 포트폴리오 이력을 조망할 수 있는 필터 스위치를 제공합니다.
- **상세 요구사항:**
  - `AssetDetailTab` 상단에 PrimeReact `RadioButton` 그룹을 배치하여 **[전체 종목] / [보유 중 종목]** 선택 구조를 만드십시오.
  - 백엔드로부터 `quantity: 0`인 데이터가 포함된 배열을 수신합니다.
  - **렌더링 규칙:**
    - [보유 중 종목] 선택 시: `quantity > 0`인 종목만 화면에 필터링하여 노출합니다.
    - [전체 종목] 선택 시: 수량이 0인 종목도 리스트에 표시하되, DataTable의 `rowClassName`을 활용해 해당 행(Row) 전체의 **CSS opacity를 0.7**로 주어 시각적으로 휴면 종목임을 명확히 구분하십시오.

## 📌 4단계. 캔들 차트 인터랙션 고도화 (기간 이동 및 전체 화면)
- **내용:** `ApexCharts` 기반의 캔들 차트 조작성을 대폭 강화합니다.
- **상세 요구사항:**
  - **기간 이동(Pan / Touch Move):**
    - 차트 영역을 마우스로 좌우 드래그하거나 모바일 기기에서 터치 무브할 때 X축 범위 이동을 감지하십시오.
    - 감지된 기간(`start_date`, `end_date`)을 바탕으로 공통 API 모듈을 사용해 백엔드 API(`GET /api/stocks/ohlcv`)를 호출하여 시계열 데이터를 갱신하십시오.
    - **🚨 [성능 방어] 300ms Debounce 결계:** 드래그 중 무차별적인 API 호출로 브라우저가 크래시되는 것을 막기 위해, API 요청 함수에 반드시 **최소 300ms 디바운스**를 적용하십시오.
  - **전체 화면(Fullscreen):**
    - 차트 컨테이너 우측 상단 영역에 미니멀한 '전체 화면 토글' 버튼을 신설하십시오.
    - 브라우저 표준 `Element.requestFullscreen()` 및 `document.exitFullscreen()` API를 결합하여 차트를 화면에 가득 채워 구현하십시오.
