# Account Daily Balance 자동 산출 (하이브리드 방식) 구현 완료

의사 결정해 주신 내용을 바탕으로 프론트엔드 로그인 호출 대신, 백엔드 코어 단에서 모든 동기화를 책임지는 **하이브리드 방식(서버 시작 시 1회 + 매일 자정 배치)**으로 일간 잔고 동기화(`account_daily_balance`) 기능을 완벽히 구현했습니다!

## 1. 구현된 핵심 아키텍처

### 🔄 하이브리드 자동 동기화 (APScheduler)
*   **Startup Sync (초기 동기화)**: FastAPI 백엔드 서버가 구동될 때 `lifespan` 이벤트를 통해 즉시 `run_midnight_batch()` 함수가 1회 실행됩니다. 개발 중이거나 서버 재시작 시점에 누락된 데이터가 자동으로 꽉 채워집니다.
*   **Midnight Batch (자정 배치)**: 서버가 24시간 연속 동작하더라도 문제없도록, `apscheduler` 라이브러리의 `BackgroundScheduler`를 도입했습니다. 매일 자정(00시 30분)에 백그라운드 스레드가 조용히 돌며 전날까지의 일간 잔고를 동기화합니다.

### 🧠 순수 영업일(Trading Days)만 취합
*   별도의 휴일 캘린더나 크롤링을 사용하지 않고, 주식 종가가 수집되어 있는 `stock_ohlcv_cache` 테이블에서 **유효한 `trade_date`**만을 `DISTINCT`로 뽑아 순회하도록 설계했습니다.
*   결과적으로 공휴일, 주말 등 시장이 열리지 않은 날짜는 자연스럽게 잔고 스냅샷 대상에서 제외됩니다.

### 💰 순입금액 기반 수익률(Return Rate) 산정
*   원금 계산 시 `DEPOSIT`은 더하고 `WITHDRAW`는 빼는 방식으로 **총 투자 원금(`total_principal`)**을 누적 산정합니다.
*   이자와 배당(`INTEREST`, `DIVIDEND`)은 현금 잔고에는 더해지지만 원금(`total_principal`)에는 반영하지 않으므로, 순수한 자산 증가(수익)로만 취급되어 수익률(`return_rate`)이 정확히 계산됩니다.

## 2. 수동 제어용 API 신설

백그라운드 처리 외에도 언제든 강제로(또는 특정 계좌만) 동기화를 트리거할 수 있도록 백엔드 API를 열어 두었습니다. (현재 프론트엔드 연동은 불필요하지만, 향후 수동 동기화 버튼 등을 만들 때 사용할 수 있습니다.)

*   `POST /api/accounts/sync-all-daily-balances`: 등록된 모든 계좌의 누락된 일자 잔고를 한 번에 동기화합니다.
*   `POST /api/accounts/{acc_cd}/sync-daily-balances`: 특정 계좌 하나만 지정하여 동기화합니다.

## 3. 남은 과제 가이드 (선택 사항)

> [!NOTE]
> **추가 권장 사항 (Cascading Recalculation)**
> 
> 현재 방식은 오늘이나 어제 같은 **최근의 누락된 일자**를 앞에서부터 차곡차곡 채우는 데(Incremental) 최적화되어 있습니다. 
> 만약 사용자가 **과거(예: 3개월 전)의 주식 매매 내역이나 입출금 내역을 뒤늦게 수정/추가**한다면, 해당 일자부터 오늘까지의 스냅샷을 통째로 덮어써야(Overwrite) 합니다.
> 
> 향후 프론트엔드에서 과거 내역 수정 기능을 완성할 경우, 백엔드의 `POST` / `PUT` / `DELETE` 트랜잭션 라우터에서 `sync_daily_balances_for_account` 함수를 호출할 때 **특정 날짜 이전의 스냅샷을 비워버리는(Delete) 로직**을 추가하면 과거 내역 수정에도 완벽하게 대응할 수 있습니다. 

수고하셨습니다! 프론트엔드 연동 없이 백엔드만으로 튼튼하게 돌아가도록 구성되었으므로, 이제 대시보드 쪽에서 자유롭게 이 데이터를 당겨서 차트나 KPI에 활용하실 수 있습니다.
