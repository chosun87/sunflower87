# 📊 주식 차트 프론트엔드 시각화 구현 및 사용자 경험 보고서 (Frontend Implementation Report)

본 문서는 기획 및 비즈니스 의도에 맞추어 구현된 **[sunflower87] 주식 캔들차트 프론트엔드(FE) 시각화 및 UX 최적화 과정**을 정리한 공유용 보고서입니다.

---

## 1. 프론트엔드 설계 철학 및 기술 스택 (Core Tech Stack)

차트의 세련된 비주얼과 모바일/태블릿을 넘나드는 완벽한 가독성을 제공하기 위해, **React-ApexCharts**를 기반으로 하며 프론트엔드 자체의 가상 상태 제어 기술을 융합하여 설계했습니다.

* **핵심 라이브러리**: `react-apexcharts`
* **주요 UI 연동**: PrimeReact 테마 변수(`var(--blue-600)`, `var(--surface-border)` 등) 연동을 통한 자동 라이트/다크 테마 스위칭 지원.

---

## 2. 핵심 구현 사항 및 기획적 가치 (Key Features & UX Value)

### ① 주말 및 휴장일 거래 공백(Trading Gap) 100% 제거
* **기획적 의도**: 주식 시장은 주말 및 공휴일에 개장하지 않으므로, 일반적인 시계열 축(datetime)을 사용하면 차트 중간중간에 거래가 없는 텅 빈 주말 공백이 노출되어 시각적 흐름이 끊기게 됩니다.
* **구현 방식**:
  - X축의 타입을 일반적인 `datetime` 대신 **`category` (카테고리) 방식**으로 선언했습니다.
  - 백엔드에서 받은 영업일 순수 데이터 리스트의 인덱스(Index)를 기반으로 1:1 강제 매핑(Sequential Mapping)을 함으로써 주말 공백을 완전히 파쇄하여 매끄럽게 연결되는 흐름을 제공합니다.

### ② 명품 Glassmorphism 커스텀 다크 툴팁 (Custom Dark Tooltip)
* **기획적 의도**: 사용자가 마우스를 올렸을 때(Hover) 차트의 기본 툴팁은 투박하여 고급 주식 플랫폼의 느낌을 주기 어렵습니다.
* **구현 방식**:
  - CSS의 `backdrop-filter`와 반투명 백그라운드를 활용한 **글래스모피즘(Glassmorphism) 스타일의 다크 테마 툴팁**을 HTML/CSS 템플릿으로 커스텀 개발했습니다.
  - 마우스를 올리면 시가, 고가, 저가, 종가는 물론 5일선 및 20일선 이동평균선(MA) 정보까지 한눈에 가독성 높게 표시하여 전문 트레이딩 뷰의 경험을 선사합니다.

### ③ React 표준 상태 기반 정석 렌더링 전환 (Standard React Rendering Workflow)
* **기획적 의도**: 과거 비동기 컴포넌트 마운트 프레임 꼬임을 방지하기 위해 임시 사용하던 `setTimeout` 20ms 편법 코드를 전면 폐기하고, React 본연의 선언적 데이터 흐름에 따라 최적의 속도로 안전하게 차트를 그립니다.
* **구현 방식**:
  - 마운트 딜레이를 제거하고 수신 완료 시 즉시 상태(State) 전이를 기동합니다.
  - 차트 드로잉 영역은 `isChartLoading` 스피너 가드 및 수신 데이터 배열의 길이(Array.length) 검증 가드에 의해서만 안전하게 통제받도록 구조를 개편하여 지연 시간 제로의 명품 속도를 자랑합니다.

### ④ 80거래일 확장 윈도우 동적 타이틀 바인딩 (Dynamic Trend Header)
* **기획적 의도**: 16시 타임윈도우 엔진 연계 고도화 스케줄에 맞추어 시계열이 60거래일에서 80거래일로 확대됨에 따라, 어떤 거래일수 데이터가 수신되더라도 깨지지 않고 반응하도록 유연성을 확보합니다.
* **구현 방식**:
  - 차트 헤더 타이틀에 하드코딩 문구 대신 실제 수신된 데이터 개수를 동적으로 맵핑하도록 `${chartData.rawList.length}거래일` 변수형 포맷터를 탑재했습니다. 
  - 이에 따라 데이터 범위 확장 등 향후 어떠한 비즈니스 스케줄 조정 시에도 차트 구성의 유연성과 유지보수성이 극대화되었습니다.

### ⑤ React 상태 기반 반응형 눈금 밀도 제어 (Perfect Responsive Grid)
* **기획적 의도**: 모바일 화면에서는 차트 가로폭이 좁아 눈금 날짜가 겹쳐서 알아볼 수 없게 되고, 큰 화면에서는 눈금이 너무 없어서 허전해 보이는 현상을 방지해야 합니다. 또한 화면을 줄였다가 다시 전체 화면으로 바꿀 때(Maximize) 레이아웃이 꼬이는 현상을 원천 방지합니다.
* **구현 방식**:
  - 브라우저 크기 조절 시 다중 중첩 버그를 일으키는 `ApexCharts` 자체의 반응형 옵션에 의존하지 않고, **React의 상태 추적 기술을 활용하여 근본적으로 버그를 해결**했습니다.
  - `window.resize` 이벤트를 실시간으로 추적하여 `windowWidth` 상태를 업데이트하고, 이에 맞춰 가장 아름다운 X축 눈금 한계값(`tickAmount`)을 단방향 실시간 계산하여 차트 옵션에 직접 주입합니다.
  - 이로 인해 모바일, 태블릿, 노트북, 듀얼 모니터 최대화 등 어떠한 해상도 변화나 브라우저 창 조정 환경에서도 **100% 무결점 정밀 반응형 렌더링**을 구현해 냈습니다.

## 3. 디바이스별 X축 눈금 조밀도 규격 (Grid Specifications)

| 디바이스 유형 (Device Type) | 기준 화면 너비 (Width Breakpoint) | 추천 눈금 개수 (tickAmount) | 기획적 기대 효과 |
| :--- | :---: | :---: | :--- |
| **모바일 (Mobile)** | `768px` 이하 | **`8`개** | 좁은 화면에서도 텍스트가 겹치지 않고 쾌적한 정보 제공 |
| **태블릿 (Tablet / Small Laptop)** | `1200px` 이하 | **`14`개** | 중형 화면 비율에 맞는 최적의 균등 간격 배치 |
| **데스크톱 (Desktop / Maximize)** | 기본 사양 | **`20`개** | 넓은 고해상도 화면에 정밀하고 풍부한 추세선 노출 |

## 4. 최종 시각적 만족도 요약 (Visual Excellence Summary)
* **테마 일치성**: 프론트엔드의 테마 컬러 가이드라인을 백퍼센트 추종하도록 캔들 양봉(`var(--blue-600)`) 및 음봉(`var(--red-600)`) 컬러를 정교하게 바인딩하여 앱의 전반적인 미려함을 극대화했습니다.
* **사용자 친화적 피드백**: 데이터 로딩 중에는 PrimeReact의 고속 스피너(`ProgressSpinner`)를, 비어 있는 경우에는 명확한 빈 화면 가이드를 보여주어 사용자가 진행 상황을 즉시 직관적으로 인지할 수 있게 유도했습니다.
