작성일시: 2026-06-02
작성자: 메인 AI (팀 대리 작성: PL, DB, BE, FE, QA)

---

## 1. 개요
최근 작업에서는 **대시보드의 실시간 시세 연동 오류 및 KPI 계산 버그 수정**, **불필요한 레거시 파일 청소**, 그리고 **일일 잔고 동기화(SyncAccountBalanceDaily) 기능의 고도화 및 네이밍 리팩토링**이 중점적으로 진행되었습니다.

---

## 2. 파트별 상세 작업 내역

### 🧩 1) PL & DB 파트
- **핵심 엔티티 명칭 확립**: `AccountBalanceDaily` (일일 잔고 데이터)라는 핵심 도메인 모델의 네이밍을 DB, BE, FE 전반에 걸쳐 일관성 있게 유지하도록 정책 수립 및 롤백 조치.
- **데이터 흐름 통제**: 캘린더 기반의 유연한 동기화와, 전체 데이터를 일괄 초기화/동기화하는 관리자(Settings) 목적의 동기화 트랙을 분리 설계.

### 🐍 2) BE (BackEnd) 파트
- **실시간 주가 반영(KPI) 로직 수정**:
  - `portfolio_service.py`: 포트폴리오 자산 가치 평가 시 `StockOHLCVCurrent` (실시간 현재가)를 최우선으로 조회하도록 수정하여, 장중 실시간 수익률이 대시보드에 즉시 반영되도록 개선.
  - `dashboard_service.py`: 당일 KPI 계산 시 과거 일자 비교 기준을 보정하여 수익률 계산 정확도 100% 확보.
- **코드 베이스 클린업**:
  - 더 이상 사용하지 않는 과거 마이그레이션 스크립트 및 테스트 파일(`check_cash_balance.py`, `update_dt.py`, `daily_balance_service.py` 등) 삭제 및 정리.
  - 실수로 삭제되었던 DB 마이그레이션 핵심 파일(`migrate.py`) 즉각 복구 및 `main.py` 모듈 참조 버그 해결.
- **API 엔드포인트 세분화 및 리팩토링**:
  - `account_balance_daily_service.py`의 핵심 함수명(`sync_account_balance_daily`) 원상 복구.
  - **계좌 동기화 API**: `POST /api/accounts/sync_balance_daily` (간결화)
  - **설정용 전체 동기화 API**: `POST /api/settings/sync_account_balance_daily` (명확화)
  - `qa_test.py` 스크립트를 변경된 API 경로에 맞추어 최신화 및 테스트 통과.

### ⚛️ 3) FE (FrontEnd) 파트
- **일일 잔고 동기화 화면 고도화 (`SyncAccountBalanceDaily.tsx`)**:
  - 캘린더 기반 기간 설정(start_date, end_date) API 연동 완료 (`/api/accounts/sync_balance_daily`).
  - **[신규] 전체 기간 동기화 기능**: 캘린더 상단에 '전체 기간 동기화' Checkbox(기본: false) 추가.
  - 체크 시 하단 캘린더를 비활성화(`disabled`) 처리하고, 동기화 버튼 클릭 시 `settings` 라우터(`/api/settings/sync_account_balance_daily`)를 호출하여 관리자 모드의 전체 동기화가 이루어지도록 UI/로직 브랜칭 구현.
- **라우팅 및 API 통신 모듈 정비**:
  - `fe/src/api/index.ts`에 2가지 분리된 API 호출 함수(`syncAccountBalanceDaily`, `syncAccountBalanceDailyAll`) 등록 완료.
  - UI 주소창 경로는 `/sync_balance_daily`로 간결하게 통일(`App.tsx`, `SidebarData.ts`).
