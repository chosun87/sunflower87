## 📌 1단계: 코어 데이터 무결성 및 종목 마스터 고도화
- **백엔드 (EARTH-BE):** - `be/schemas.py`의 `TransactionCreate` 스키마 내 `quantity`(수량)와 `price`(단가) 필드에 `Field(..., gt=0)` 제약 조건을 강제하여 무결성을 방어합니다.
  - `cache_stocks` 테이블에 `is_active` (BOOLEAN, 디폴트 True) 컬럼을 신설합니다. 서버 기동(Lifespan) 및 신설될 수동 동기화 API 호출 시 `pykrx`를 통해 주식 및 ETF 목록을 통합 수집(ETF는 `market='ETF'`)하고 로컬 DB와 대조(Diff)하여 상폐 종목을 `is_active = False`로 전환(Soft Delete)합니다.
  - 온디맨드 수동 동기화 엔드포인트(`POST /api/stocks/sync-master`)를 신설하여, 충돌 시 무조건 최신 정보로 덮어쓰는 Upsert(`INSERT OR REPLACE`) 메커니즘을 강제하고 갱신 전 기존 데이터를 완전히 비우는(Delete All) 안전장치를 구축합니다.
- **프런트엔드 (EARTH-FE):** - 대시보드 우측 상단에 '🔄 종목 마스터 최신화' 버튼을 신설합니다. 통신 중에는 `disabled` 및 로딩 상태를 유지하며 완료 시 PrimeReact `Toast` 알림을 표출합니다. 기존 송출 전 `Math.abs()` 부호 무결성 결계는 현행 유지합니다.

## 📌 5단계: 캔들 차트 인터랙션 고도화 및 프리페치 뷰포트 빌드
사용자 조작 시 잦은 API 요청으로 인한 브라우저 버벅임을 원천 방어하기 위해 '160일 데이터 선수집(Pre-fetch)' 및 '80일 화면 표출(Viewport)' 규격을 도입하는 대규모 고도화 단계입니다.

### 1. 백엔드 (EARTH-BE) 조치 사항 (240-Day Buffer 서빙)
* **API 파라미터 및 버퍼 확장:** `GET /api/stocks/ohlcv` API가 프런트엔드의 요청 기간(`start_date`, `end_date`)을 수용하도록 확장합니다.
* **🚨 240일 버퍼 규격 강제:** 프런트엔드가 화면에 160일치 시계열을 렌더링하고 그 위에서 20일 이동평균선(20MA) 및 5일 이동평균선(5MA)을 끊김 없이 연산할 수 있어야 합니다. 따라서 백엔드는 프런트엔드가 요청한 `start_date` 시점보다 **최소 240거래일 앞선 과거 데이터부터 누적하여 오름차순(ASC)으로 정렬**한 뒤 페이로드 선두에 결합(Prepend)하여 서빙해야 합니다. 이를 위해 동적 날짜 범위를 `target_start_date = start_date - 360일` 수준으로 넉넉하게 확장 계산하여 DB를 조회합니다.
* **Historical Backfill 자가치유 보강:** `sync_ohlcv_cache` 로직이 지정된 과거 시점의 데이터를 체크하여 DB에 부족한 경우, 실시간으로 외부망(`pykrx`)을 호출해 캐시 테이블 앞단을 점진적으로 채워 넣는 유동형 스케일링 캐싱을 집행합니다.

### 2. 프런트엔드 (EARTH-FE) 조치 사항 (160일 페치 & 80일 표출)
* **초기 데이터 선수집 (Pre-fetch):** 특정 종목 진입 시 초기 로딩 단계에서 백엔드 확장 API를 통해 총 **160거래일치**의 OHLCV 시계열 데이터를 한 번에 가져옵니다.
* **ApexCharts 데이터 주입 및 뷰포트 강제 제한:** - 가져온 160일치 데이터를 차트 `series` 배열에 **전부 밀어 넣습니다.**
  - 차트 옵션의 **`xaxis.min`과 `xaxis.max` 속성을 동적으로 제어하여 초기 화면 표시 영역(Viewport)을 '최근 80거래일' 구간으로 제한하여 렌더링**합니다.
  - 이 설정을 통해 사용자가 마우스 드래그(Pan) 또는 모바일 터치 무브로 차트를 왼쪽으로 밀 때, **추가적인 API 통신 없이 이미 메모리에 로드된 과거 80일치 데이터가 60fps 수준으로 지연 시간(Lag) 없이 아주 매끄럽게 표출**됩니다.
* **경계선 인식 기반 백필 트리거:** 사용자가 스크롤을 강하게 하여 이미 로드된 160일 데이터의 좌측 최끝단(과거 경계선)에 도달했을 때만, 차트 기간 범위(`min`, `max`)를 새로 계산하여 백엔드에 다음 과거 데이터를 요청하도록 이벤트를 제어합니다.
* **🚨 성능 방어 및 로딩 오버레이 결계:** - 경계선 도달로 인한 과거 데이터 추가 요청 시 브라우저 크래시를 막기 위해 반드시 **최소 300ms 디바운스(Debounce)**를 적용합니다.
  - 디바운스가 끝난 직후 백엔드가 외부망에서 과거 데이터를 실시간 스크래핑하는 동안 화면 멈춤 오해를 차단하기 위해, 통신이 완료될 때까지 **차트 전면에 은은한 로딩 스피너 혹은 블러(Blur) 오버레이 레이어**를 노출합니다.
* **Native Fullscreen API 연동:** 차트 상단 우측에 미니멀한 '전체 화면 토글' 버튼을 배치하고, 브라우저 표준 `requestFullscreen()` 및 `exitFullscreen()` 인터랙션을 빌드하여 화면 리사이즈 300ms 디바운스 로직과 결합합니다.
