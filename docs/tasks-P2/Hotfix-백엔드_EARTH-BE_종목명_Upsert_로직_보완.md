# 🌍 [Hotfix] 백엔드 EARTH-BE(어씨베) 종목 마스터 Upsert 무결성 보완 지시서

- **작성자:** 기획자 MOON(무니)
- **수신자:** 백엔드 개발자 EARTH-BE(어씨베)
- **프로젝트 명:** sunflower87 통합 자산 관리 대시보드 고도화
- **이슈 항목:** 온디맨드 수동 동기화를 가동해도 종목코드 396500이 'TIGER 반도체TOP10'으로 갱신되지 않고 'KBSTAR Fn수소경제테마'로 고정되어 출력되는 현상 방어

---

## 🚨 원인 분석 (Root Cause)
- **현상:** 수동 동기화 API(`POST /api/stocks/sync_master`)를 호출하여 외부망(`pykrx`)에서 올바른 데이터를 받아왔음에도 로컬 DB 값이 바뀌지 않습니다.
- **원인:** 종목 마스터 캐시 테이블(`cache_stocks`)에 데이터를 적재할 때, **"종목코드가 이미 존재하면 업데이트를 스킵(Skip/Ignore)하는 로직"**으로 구현되어 있습니다. 과거 인덱스 밀림이나 테스트 더미 데이터로 인해 한 번 잘못 매핑된 데이터(`396500` = 수소경제)가 상위 무결성 제약조건에 막혀 최신 올바른 데이터로 덮어써지지(Update) 못하는 상태입니다.

---

## 🛠️ 조치 지시 사항 (Action Items)

### 1. 덮어쓰기(Upsert) 메커니즘 강제 적용 (코어 코드 수정)
- `cache_stocks` 테이블에 데이터를 밀어 넣는 수집/갱신 코어 함수를 수정하십시오.
- 단순히 존재 여부를 체크하고 넘어가거나 `INSERT OR IGNORE` 구조를 사용하지 말고, **무조건 최신 정보로 덮어쓰는 Upsert(지정 충돌 시 업데이트) 구조**를 적용해야 합니다.
- **SQLite / SQLAlchemy 구현 가이드:**
  - SQLAlchemy Core의 `insert(table).values(...)`에 `.on_conflict_do_update()`를 체이닝하여 충돌 시 `stock_name`, `market`, `is_active` 등의 컬럼을 외부 수집 데이터(`EXCLUDED`)로 강제 갱신하도록 구성하십시오.
  - 또는 SQLite Native 구문인 `INSERT OR REPLACE INTO cache_stocks ...` 규칙을 쿼리에 강제하십시오.

### 2. 오염된 기존 마스터 데이터 초기화 (One-time Cleanup)
- 현재 테이블 내에 396500 외에도 얼마나 많은 종목코드가 밀려서 꼬여있는지 추적이 불가능합니다.
- 따라서 이번 핫픽스 적용 배포 시점 또는 수동 동기화 로직의 시작 서두에 **`cache_stocks` 테이블의 기존 레코드를 완전히 비워버리는(Truncate / Delete All) 로직을 일시적으로 수행하거나 초기화 스크립트를 가동**하십시오.
- *주의:* `cache_stocks`는 단순 이름표 캐시 테이블이므로 전체 삭제 후 외부망 재수집을 돌려도 매매 원장(`transactions`)이나 자산 현황 역산에는 아무런 영향을 주지 않으므로 안전합니다.

### 3. 무결성 정합성 최종 검증
- 수정 후 수동 동기화 버튼을 재가동하여, DB 내부 및 검색 API 응답에서 `396500` 코드가 공식 명칭인 **'TIGER 반도체TOP10'**으로 완벽하게 치환되어 출력되는지 육안 및 디버그 콘솔로 검증한 후 커밋하십시오.
