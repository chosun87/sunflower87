# 📊 Phase-2 추가 과제: 온디맨드 수동 동기화 (On-Demand Sync) 버튼 신설 구현 계획

본 문서는 Phase-2 개발 진행 중 추가된 **'온디맨드 수동 동기화(On-Demand Sync) 기능'**에 대한 단독 기술 규격 및 작업 지시서입니다. 1인 VIP 환경의 최적화 철학을 유지하면서 종목 데이터의 최신성을 수동으로 확보할 수 있도록 설계되었습니다.

---

## 📌 1. 개요 및 비즈니스 로직
- **배경:** 현재 시스템은 서버 기동 시(Lifespan)에만 외부망(`pykrx`)에서 최신 종목 및 ETF 목록을 수집하여 마스터 테이블(`cache_stocks`)을 채웁니다. 이로 인해 서버를 장시간 켜두었을 때 신규 상장 종목이 누락될 수 있는 문제를 해결하고자 합니다.
- **솔루션:** 로그인 속도를 느리게 하거나 무거운 백그라운드 배치를 상시 가동하는 대신, 사용자가 필요할 때 직접 대시보드에서 갱신을 트리거할 수 있는 **온디맨드(On-Demand) 버튼**을 신설합니다.

---

## 📌 2. 개발자별 상세 작업 지시

### 🌍 백엔드 개발자 (EARTH-BE) 조치 사항

#### 1. API 엔드포인트 신설
- **경로:** `POST /api/stocks/sync-master`
- **기능:** 호출 시 외부망(`pykrx`)의 최신 주식 및 ETF 데이터를 동적으로 수집하여 로컬 DB의 종목 마스터를 갱신합니다.

#### 2. 동기화 코어 로직 매핑
- 기존 `lifespan` 기동 시점에 실행되던 종목 세팅 로직을 함수화하여 이 엔드포인트에 연결하십시오.
- **수집:** `pykrx.stock.get_market_ticker_list(date)` 및 `pykrx.stock.get_etf_ticker_list(date)` 통합 수집 (ETF 종목은 `market='ETF'` 명시).
- **대조 및 Soft Delete:** 최신 수집 데이터와 로컬 DB 데이터를 대조(Diff)하여, 오늘 상장 상태가 확인되지 않는 종목은 `is_active = False`로 상태를 전환하는 자가 치유를 수행해야 합니다.

---

### 📊 프런트엔드 개발자 (EARTH-FE) 조치 사항

#### 1. UI 컴포넌트 배치
- **위치:** 대시보드 우측 상단 영역 또는 글로벌 헤더/컨트롤 바
- **형태:** PrimeReact 버튼 컴포넌트를 활용하여 **'🔄 종목 마스터 최신화'** (또는 미니멀한 새로고침 아이콘 형태) 버튼을 배치하십시오.

#### 2. 상태 관리 및 인터랙션 (Performance Guard)
- **로딩 상태 제어:** 버튼을 누르면 API 응답을 받기 전까지 버튼을 `loading={isLoading}` 상태로 전환하고 **비활성화(Disabled)** 처리하여 무차별적인 중복 호출을 원천 차단하십시오.
- **성공/실패 피드백:** 공통 API 모듈(`fe/src/api/index.js`)을 통해 통신을 수행한 뒤, 동기화가 성공적으로 완료되면 PrimeReact `Toast` 컴포넌트를 사용하여 *"종목 마스터 데이터가 최신 상태로 동기화되었습니다."*라는 안내 문구를 화면에 띄우십시오.

## 📌 3. 수동 검증 계획 (Verification Plan)

1. **중복 클릭 방지 검수:** 대시보드에서 '🔄 종목 마스터 최신화' 버튼을 클릭했을 때, 즉시 로딩 애니메이션이 활성화되며 버튼 클릭이 막히는지 확인합니다.
2. **네트워크 통신 검수:** 브라우저 개발자 도구(F12) 네트워크 탭에서 `POST /api/stocks/sync-master` 요청이 정상적으로 송출되고 `200 OK` 응답이 떨어지는지 확인합니다.
3. **사용자 피드백 검수:** 응답 완료 즉시 로딩 상태가 해제되면서 화면 우측 상단 등에 성공 Toast 알림 팝업이 무결하게 표출되는지 검증합니다.
4. **최종 정합성 검수:** 갱신 완료 후 종목 검색창(`AutoComplete`)을 열어 최신 ETF 및 주식 종목이 정상적으로 필터링 및 조회되는지 확인합니다.
