# Phase 5 백엔드 설계 의견 (BE)

## 1. 기획안에 대한 BE 파트 의견
기존에 `account.py` 라우터에 구현된 `/sync-all-daily-balances` 와 `/sync-daily-balances` 엔드포인트를 기획 내용에 맞춰 고도화하겠습니다. 기간을 입력받아 부하를 줄이는 것은 필수적인 성능 개선 포인트입니다.

## 2. API 수정 방향
* **라우터 수정 (`routers/account.py`):**
  - POST `/api/accounts/sync-all-daily-balances` 및 POST `/api/accounts/{acc_cd}/sync-daily-balances` 에 Payload 로 `start_date`와 `end_date`를 선택적으로 받도록 스키마(`schemas.py`)를 수정합니다.
  - 파라미터가 없으면 기본값으로 '한 달 전 ~ 어제'를 처리하도록 로직을 구현합니다.
* **비즈니스 로직 수정 (`services/daily_balance_service.py`):**
  - 기존에는 `sync_daily_balances_for_account` 함수가 전체 기간을 돌았다면, 이제는 인자로 전달받은 기간 동안만 `Transaction`, `TransactionCash`, `StockOHLCVCache`를 참조해 과거 EOD 자산을 추적/계산합니다.
  - `end_date`가 미래이거나 '오늘'인 경우, '어제' 날짜까지만 계산되도록 방어 코드를 적용합니다.
  - **계좌개설일(dt_opened) 하한 방어:** 대상 계좌의 개설일 이전 날짜로는 동기화 연산이 수행되지 않도록 `start_date`를 `dt_opened` 기준으로 잘라내는 하한선(Bound) 방어 로직을 추가합니다.

## 3. 당일 데이터 실시간 제공
`account_daily_balance`에는 어제까지만 데이터가 있으므로, 당일의 잔고와 수익률은 기존의 `/api/accounts/{acc_cd}/performance`나 계좌 요약 API 호출 시 실시간으로 계산하여 던져주도록 분리 유지합니다.
