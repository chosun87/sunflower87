## 2. Walkthrough
1. **페이지 리팩토링 및 라우팅 수정:**
   포괄적이었던 '데이터 관리' 역할을 명확히 하기 위해 `SyncDailyBalance` 페이지로 역할을 축소하고, 관련 라우팅 주소와 사이드바 메뉴 정보를 업데이트했습니다.
2. **DatePicker 기간 설정 적용 (UI 방어 및 BE 위임):**
   PrimeReact의 `Calendar` 컴포넌트를 활용하여 시작일(`start_date`)과 종료일(`end_date`)을 선택할 수 있도록 range 모드 및 `locale="ko"`로 구현했습니다.
   * **초기값 설정:** 페이지 접속 시 시작일은 '1개월 전', 종료일은 '어제'로 자동 설정됩니다.
   * **사용자 실수 방지:** 종료일 Calendar의 `maxDate` 속성에 '어제' 날짜 객체를 할당하여 오늘 이후의 날짜를 선택할 수 없도록 원천 차단했습니다.
   * **계좌개설일 연동 위임:** 모든 계좌 일괄 동기화 방식이므로 프론트엔드 UI의 `minDate` 제한은 해제하고, 하한선 제어는 백엔드의 계좌별 방어 로직으로 위임했습니다.
3. **API Payload 연동:**
   동기화 버튼 클릭 시, 선택된 날짜를 `YYYY-MM-DD` 문자열 포맷으로 파싱하여 API Body에 실어 백엔드로 전송하도록 구현을 완료했습니다.
