# Phase 5 완료 보고서

## 1. 개요
* **진행 기간:** Phase 5
* **주요 목표:** 과거 데이터 수정에 따른 일일 잔고 동기화 기능 구현 및 데이터 무결성 검증, 캘린더 UX 개선
* **결과:** 모든 요구사항 구현 및 치명적 결함 조치 완료, 통합 테스트 최종 통과

## 2. 주요 산출물 및 변경 내역

### 2.1. 데이터베이스 및 백엔드 (BE)
* **스키마 변경:** `account` 테이블에 계좌 개설일(`dt_opened`) 컬럼 추가 (적용 완료 경로: `db/sunflower87.db`)
* **마이그레이션 방어:** `migrate.py` 스크립트 실행 시 기존 계좌의 `dt_opened` 데이터가 초기화되지 않고 안전하게 백업 및 복원되도록 패치 적용.
* **잔고 동기화 API (`/api/accounts/sync_account_daily_balance`):**
  * 지정된 기간(`start_date`, `end_date`) 내의 기존 `account_daily_balance` 레코드를 삭제 후 거래내역 기반으로 재계산하여 `INSERT`하는 최적화 로직 구현.
  * **하한/상한 방어 로직:** `start_date`가 계좌개설일 이전일 경우 개설일로 보정하고, `end_date`가 오늘/미래일 경우 어제 날짜로 강제 치환하여 데이터 오염을 완벽히 방지함.
* **내역 조회 API 필터링 버그 수정:**
  * 주식 매매 내역 및 계좌 입출금 내역 조회 시 프론트엔드의 날짜 필터(`start_date`, `end_date`)가 무시되던 버그를 수정하여 정상 필터링 되도록 조치.

### 2.2. 프론트엔드 (FE)
* **일일 잔고 동기화 화면 (`SyncDailyBalance.tsx`):**
  * 날짜 선택(Calendar) UI를 Range 모드로 구성하고 한국어(`locale="ko"`) 적용.
* **사용자 편의성(UX) 개선 (Calendar Auto-close 적용):**
  * 달력에서 시작일과 종료일(range) 선택이 완료되는 순간 오버레이가 자동으로 닫히도록 `calendarRef.current?.hide()` 로직 적용.
  * 일일 잔고 동기화 화면뿐만 아니라 **주식 매매 내역(`TransactionStockCmpt.tsx`)**, **계좌 입출금 내역(`TransactionCashCmpt.tsx`)**의 검색 캘린더에도 일괄 적용하여 일관된 UX 제공.

### 2.3. 기획 및 QA
* **문서 현행화:** 작업 과정에서 발생한 테이블 스펙 변경 및 방어 로직 상세 설계 내역을 기획/설계 문서(P-5 시리즈)에 모두 추적 기록.
* **통합 테스트 및 결함 관리:** 
  * 로컬 개발 DB와 실제 런타임 타겟 DB(`db/sunflower87.db`) 간의 스키마 불일치로 발생한 서버 크래시(Crash) 이슈를 조기 발견하여 격리 및 해결.
  * 정적 분석에 의존하던 테스트 보고 절차를 타파하고, 스크립트 및 API 호출을 통한 실제 통합 테스트(Integration Test)로 최종 PASS 처리.

## 3. 향후 과제 및 논의 사항
* **실시간 자산과 누적 자산 뷰 분리:** 일괄 동기화를 통해 EOD(End of Day) 잔고의 무결성이 확보되었으므로, 차기 Phase에서는 대시보드 화면 상에서 '실시간 평가 자산' 영역과 '과거 누적 추이 차트' 영역을 시각적으로 명확히 분리하여 사용자가 인지하기 쉽도록 UI/UX를 고도화할 필요가 있습니다.
