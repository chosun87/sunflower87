# Phase 5 프론트엔드 설계 의견 (FE)

## 1. 기획안에 대한 FE 파트 의견
엔드포인트 및 메뉴 라우팅 이름이 훨씬 명확해졌습니다. `account_balance_daily`의 목적에 맞춰 프론트엔드 UI를 구성하겠습니다.

## 2. UI/UX 변경 및 설계
* **Sidebar 라우팅 (`SidebarData.ts`) 수정:**
  - `data: '/dataManagement'` -> `data: '/sync_balance_daily'` (이름 직관화)
* **재계산 컴포넌트 화면 구성:**
  - 기간 동기화를 위한 페이지/모달 내에 'Date Picker' 컴포넌트를 배치합니다.
  - **기본값 로직:** 컴포넌트 마운트 시 `start_date`는 오늘 기준 1개월 전, `end_date`는 어제로 State를 자동 세팅합니다.
  - **계좌개설일 연동 (FE/BE 협력):** 현재 '모든 계좌 일괄 동기화'를 수행하므로, 각 계좌마다 다른 개설일(`dt_opened`)을 하나의 Date Picker `minDate`로 제한하는 것은 불가능합니다. 따라서 프론트엔드에서는 과거 기간 선택을 자유롭게 열어두고, 실제 하한선 제어는 **백엔드(BE)의 계좌별 방어 로직**에 전적으로 위임합니다.
  - '동기화 시작' 버튼을 누르면 BE의 `/api/accounts/sync_balance_daily` (또는 단일 계좌 동기화 API)를 호출하고, 완료 전까지 로딩 UI를 제공합니다.

## 3. 실시간 PKI 표출 대응
계좌 현황 대시보드를 보여줄 때 '어제까지의 자산 추이 차트'는 `account_balance_daily` 데이터를 활용하고, '오늘 실시간 자산 및 손익'은 별도로 당일 실시간 쿼리용 데이터를 Fetch 하여 UI에 결합하여 보여주는 형태로 구현하겠습니다.
