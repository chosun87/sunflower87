# P-6-01-4FE-설계 의견 (계좌 관리 기능)

## 1. UI/UX 구현 계획
기획안에서 요구한 대로, **주식 매매 내역(Trade History)** 및 **계좌 입출금 내역(Cash History)** 화면과 동일한 레이아웃과 UX 패턴을 적용합니다.

### 1.1 메인 화면 (계좌 목록 - Account Management)
* **경로**: `/accounts`
* **Layout**: 기존 내역 페이지들과 동일한 `Card`, `DataTable` 프레임 사용.
* **상단 툴바**:
  * 좌측: '새 계좌 등록' 버튼 (기존 내역 페이지의 '등록' 버튼 스타일과 동일, PrimeReact `Button` 사용)
  * 우측: 글로벌 검색 창 또는 필터 (필요시)
* **데이터 테이블**: 
  * 계좌코드, 계좌명, 금융사명, 현금 잔고, 개설일, 관리(수정/삭제 버튼) 컬럼으로 구성.
  * 페이징(Paginator) 및 정렬(Sort) 기능 활성화.

### 1.2 등록 및 수정 (모달 Dialog)
* 기존 내역 등록 화면에서 사용한 `Dialog` 컴포넌트(모달) 방식을 그대로 재사용.
* **입력 폼**: 
  * 계좌코드 (`InputText`, 수정 시에는 `disabled=true`)
  * 계좌명 (`InputText`)
  * 금융사명 (`InputText` 또는 `Dropdown`)
* 유효성 검사 실패 시 PrimeReact `Toast`를 이용한 알림 표시 (기존 시스템과 동일).

### 1.3 삭제 프로세스
* '삭제' 버튼 클릭 시, 바로 삭제하지 않고 PrimeReact의 `confirmDialog`를 띄워 "해당 계좌를 비활성화하시겠습니까?" 와 같은 경고창을 표시합니다.
* 승인 시 BE API의 `DELETE` 엔드포인트를 호출하고 화면을 갱신합니다.

## 2. 향후 작업 내용
1. `src/pages/AccountManagement.tsx` 컴포넌트 신규 생성.
2. `src/api/accountApi.ts` 에 CRUD API 호출 함수 작성.
3. `src/data/SidebarData.ts` 에 메뉴('계좌 관리') 추가.
4. 기존 공통 컴포넌트(`Toolbar`, `DataTable`)의 CSS 클래스를 재사용하여 통일성 확보.
