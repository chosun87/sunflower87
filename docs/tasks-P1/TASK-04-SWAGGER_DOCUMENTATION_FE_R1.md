## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. Swagger UI를 활용한 연동 및 테스트 프로토콜
- **인터페이스 스캔:**
  - 개발 서버 구동 후 브라우저로 `http://localhost:8000/docs` 주소에 상시 접속하여 백엔드 모델 규격과 패스 파라미터 구조를 파악하라.
- **실전 샌드박스 테스트:**
  1. `POST /api/transactions/add` 항목을 펼치고 우측 상단의 **[Try it out]** 버튼을 클릭하라.
  2. 에디터 창에 나오는 JSON 예시 데이터를 수정하여, 수동으로 매매를 찔러 넣어본 후 `Response 200` 코드가 오는지 확인하라.
  3. 통신이 성공하면 메인 대시보드 화면으로 이동하여 데이터 테이블에 해당 거래가 정상 바인딩되는지 크로스 체크를 완료하라.
- **통신 에러 처리 사양 파악:**
  - Swagger 문서 하단에 명시된 `400 Error` 구조를 기반으로, 프런트엔드 Axios 인터셉터(`Axios Interceptor`) 등에서 백엔드가 던지는 "예수금 부족" 등의 한글 메시지를 PrimeReact `Toast` 컴포넌트로 사용자 화면에 수려하게 띄워주는 예외 처리 코드를 설계하라.
