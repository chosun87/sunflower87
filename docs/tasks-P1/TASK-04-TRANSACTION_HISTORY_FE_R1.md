## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. 주식계좌 선택 Dropdown 및 예수금(Cash Balance) 배너 배치
- **Dropdown 컴포넌트 세팅:** `GET /api/accounts` API 데이터와 연동하되, `optionLabel="acc_nm"`, 선택 시 바인딩될 실제 Value는 **`optionValue="acc_cd"`**로 선언하라.
- **예수금 실시간 표기:** [보유 자산 상세] `DataTable` 상단 영역에 카드 컴포넌트를 신설하고, 백엔드가 리턴해 준 계좌별 현금 잔고인 **`cash_balance`** 필드 값을 천 단위 콤마 포맷(`toLocaleString()`)을 적용하여 출력하라.

### 3. 매매 거래 등록 모달 전송 및 더블 클릭 방지
- **API 호출 규격 준수:** 사용자가 매매 입력을 마치고 [등록] 처리 시, 계좌 식별 파라미터명을 반드시 **`acc_cd`**로 명시하여 `POST /api/transactions/add` 바디에 전송하라.
- **제출 프로세스 로딩 바인딩:** 신규 종목 매수 시 백엔드가 내부적으로 60일치 주가를 수집하는 연산을 수행하므로, 응답이 올 때까지 등록 버튼에 `loading={isSubmitting}` 상태를 주어 중복 제출을 차단하라.

### 4. 보유 자산 상세 DataTable 연산 및 컬러 브레이크 규칙
- **[현재가] 컬럼:** 백엔드가 16시 타임윈도우 기준으로 동적 서빙해 주는 현재가(`current_price`) 필드를 그대로 바인딩하라.
- **[평가금액] 컬럼:** `quantity` × `current_price`
- **[수익률] 컬럼:** `((current_price - avg_price) / avg_price) * 100` 
  - 연산 결과가 양수(+)이면 **Red 톤**, 음수(-)이면 **Blue 톤** 글자색 스타일을 입혀라. (소수점 아래 둘째 자리 고정 `.toFixed(2)`)

### 5. 전면적 데이터 리프레시 (Refetch)
- 거래 등록, 수정, 삭제 모달이 닫히는 즉시 상위 대시보드의 자산 데이터 조회 함수(`fetchAccountData()`)를 강제 호출하여 현금 잔고 뷰와 상/하단 테이블이 동시에 실시간 동기화되도록 연동하라.
