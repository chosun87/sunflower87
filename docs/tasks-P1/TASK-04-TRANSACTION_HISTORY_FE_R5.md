## 📌 [COMMON] 공통 요구사항
- **연산 매커니즘 전면 전환 (★핵심):** 총 매수금액을 프런트 단에서 `보유수량 × 평단가`로 임의 연산 시 단수 오차가 발생하므로, 백엔드가 제공하는 무결성 절대금액 필드를 그대로 가져다 바인딩하도록 화면 수식을 정정한다.
- **글로벌 파이낸스 디자인 톤 적용:** 주가 상승/평가익/매수는 파란색(`var(--blue-600)`), 주가 하락/평가손/매도는 빨간색(`var(--red-600)`)으로 컬러 브레이크를 일관되게 적용한다.
- **가시성 통제 규칙:** 수량과 금액은 우측 정렬(`align-right`), 숫자형 컬럼은 고정폭 폰트 클래스(`.monospace`)를 엄격하게 바인딩한다.

## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. [탭 1: 보유 자산 상세] DataTable 무결성 연산 맵핑 수정
보유 자산 상세 그리드의 9개 컬럼에 대해, 백엔드가 정산하여 던져준 데이터 명세를 아래 공식에 맞춰 1:1 결합하라. 임의의 2차 가공 연산을 전면 금지한다.

* **컬럼 구성 및 매핑 사양:**
    1. **종목코드** (`code`) : `align-left`, `.monospace`
    2. **종목명** (`name`) : `align-left`
    3. **보유수량** (`quantity`) : `text-align: right`, `.monospace`, 천 단위 콤마
    4. **현재가** (`current_price`) : `text-align: right`, `.monospace`, 천 단위 콤마
    5. **매입평단가** (`avg_price`) : `text-align: right`, `.monospace`, 천 단위 콤마, 소수점 아래 무조건 절사 표기
    6. **총 평가금액** (`total_eval_amt`) : `text-align: right`, `.monospace`, 천 단위 콤마
    7. **총 매수금액** (`total_purchase_amt` ★기존 화면 연산 방식에서 백엔드 수령 방식으로 변경) : `text-align: right`, `.monospace`, 천 단위 콤마
    8. **총 평가손익** (`total_profit_loss`) : `text-align: right`, `.monospace`, 천 단위 콤마
       - 수치가 **플러스(+)이면 `color: var(--blue-600)`**, **마이너스(-)이면 `color: var(--red-600)`** 적용
    9. **수익률** (`return_rate`) : `text-align: right`, `.monospace`, `.toFixed(2)` 제한
       - 수치가 **플러스(+)이면 `color: var(--blue-600)`**, **마이너스(-)이면 `color: var(--red-600)`** 적용

### 3. [탭 2: 매매 내역 히스토리] DataTable 및 수동 등록 폼 (R4 규격 승계)
- 날짜, 시간, 수량, 단가, 금액, 세금+수수료(`tax_fee`) 컬럼에 `.monospace` 및 우측 정렬 속성을 일관되게 부여하라.
- 매수(BUY)는 파란색 배지, 매도(SELL)는 빨간색 배지로 마킹하라.
- 매매 수동 등록 모달 창에 세금+수수료(`tax_fee`) 입력 컴포넌트와 계좌 키 표준명 `acc_cd` 송출 프로토콜을 유지하라.
