## 📌 [COMMON] 공통 요구사항 (데이터 예외 처리 강화)
- 사용자가 전량 매도하거나 보유하지 않은 종목을 매도 입력할 때, DB에 마이너스(-) 잔고가 쌓이거나 정합성이 무너지는 예외 상황을 원천 방지하는 안전장치를 백엔드 쿼리 단에 추가한다.

## 🌍 [BE_TASK] 백엔드 상세 구현 지침
- **매도 시 잔고 제로(0) 처리 제어:**
  - `POST /api/transactions/add` 로직 내부에서 `type == 'SELL'` 요청 처리 시, 먼저 `stocks` 테이블의 현재 보유 수량을 조회하라.
  - **케이스 A (전량 매도):** 이번 매도 수량이 보유 수량과 정확히 일치할 경우, `stocks` 테이블에서 해당 종목 레코드를 업데이트하지 말고 **`DELETE` 문으로 완전히 삭제** 처리하라.
  - **케이스 B (과매도 오류):** 보유 수량보다 매도 요청 수량이 더 클 경우, DB 처리를 중단하고 `400 Bad Request` 에러와 함께 *"보유 수량이 부족합니다."*라는 메시지를 프런트엔드로 리턴(Rollback)하라.
