## 🌍 [BE_TASK] 백엔드 상세 구현 지침

### 1. 자산 평가 API (`GET /api/accounts`) 정산 공식 전면 개정
어띠베는 자산 데이터 가공 시 화면에 표기되는 매입평단가를 기준으로 총 매수금액을 역산하는 로직을 절대 금지하고, 아래의 **무결성 정산 표준 공식**을 적용하여 JSON 응답을 구성하라.

* **총 매수금액 (`total_purchase_amt`) 규격 신설 (★핵심):**
    - 공식: `해당 종목의 모든 매수 거래 (수량 × 단가) 총합 + 해당 거래들의 tax_fee 총합`
    - 정산된 절대 원화 금액을 연산 결과물로 계산하여 응답 Body에 포함시켜라.
* **총 평가금액 (`total_eval_amt`):** `보유수량 × 현재가(16시 기준 동적 종가)`
* **총 평가손익 (`total_profit_loss`):** `총 평가금액 - 총 매수금액`
* **매입평단가 (`avg_price`):** `총 매수금액 / 보유수량` (반올림/절사 없이 순수 float 상태로 송출)
* **수익률 (`return_rate`):** `(총 평가손익 / 총 매수금액) × 100`

### 2. SQLite 거래 원장 테이블 스키마 (`transactions`)
- `transactions` 테이블의 세금 및 수수료 컬럼 `tax_fee` (INTEGER, DEFAULT 0) 및 매매 등록 API(`POST /api/transactions/add`)의 데이터 인서트 바인딩 구조를 견고히 유지하라.

### 3. 계좌 마스터 조회 API 데이터 매핑 및 16시 타임윈도우 유틸리티
- `GET /api/accounts` API 응답 Key명은 DB 컬럼명과 동일하게 `acc_cd`, `acc_nm`, `acc_company_nm`, `cash_balance`로 출력하라.
- `get_exact_trade_date_limits()` 유틸리티 함수를 가동하여 16시 이전에는 전일 종가, 16시 이후에는 당일 확정 종가를 기점으로 정확히 60거래일치 행을 확보하라.
