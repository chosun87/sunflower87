## 📌 [COMMON] 공통 요구사항
- **실시간 UI 리프레시:** 사용자가 매매 등록 폼(모달)에서 신규 거래를 입력하고 [등록] 버튼을 누르면, 대시보드 메인 화면의 **[보유 자산 상세 테이블]**과 현금 잔고 뷰가 화면 깜빡임 없이 즉시 최신 데이터로 동기화되어야 한다.

---

## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. 주식계좌 선택 Dropdown 및 Calendar 한국어 패치
- `Dropdown` 컴포넌트를 배치하고 `GET /api/accounts` API 데이터와 연동하라. (`optionValue="acc_cd"`, `optionLabel="acc_nm"`)
- Calendar 컴포넌트에 `locale="ko"` 속성을 부여하여 요일/월이 한국어로 렌더링되게 하라.

### 3. ★ 매매 등록 후 보유 자산 상세 화면 동기화 프로토콜
- 사용자가 선택한 계좌 데이터 및 매매 스펙을 **`acc_code`** 파라미터명에 실어서 `POST /api/transactions/add`로 전송하라.
- **상태 체인 바인딩 (Data Refetch):**
  - 거래 등록(또는 수정/삭제) API가 성공 응답(`status: success`)을 반환하는 즉시, 팝업 Dialog를 닫고 상위 대시보드 컴포넌트가 소유한 자산 조회 API 데이터 갱신 함수(`fetchAccountData()`)를 강제로 재트리거하라.
  - 이를 통해 메인 대시보드 상단의 **[보유 자산 상세] `DataTable`**과 하단 [거래 내역 히스토리] `DataTable`이 동시에 리프레시되어 백엔드 SQLite DB의 최신 자산 상태를 즉각 반영하도록 설계하라.
