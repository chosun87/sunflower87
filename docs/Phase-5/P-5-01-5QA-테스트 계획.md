# Phase 5 품질 보증 설계 의견 (QA)

## 1. 기획안에 대한 QA 파트 의견
테이블명이 `account_daily_balance`로 정정되었고, 기존 시스템에 있는 테이블을 활용하는 만큼 회귀 테스트(Regression Test)와 데이터 정합성 확인에 초점을 맞추겠습니다.

## 2. 주요 테스트 케이스 (TC)
* **TC-01: 재계산 시 기본 기간 적용 여부**
  - FE에서 아무 옵션을 건드리지 않고 버튼을 눌렀을 때, BE 로그 상 1개월 전부터 어제까지로 Parameter가 맵핑되어 실행되는지 확인.
* **TC-02: 사용자 임의 기간 설정 정합성**
  - 특정 과거 기간(예: 3개월 전 ~ 2개월 전) 지정 시, 그 기간 내의 매매 내역 및 현금 입출금 내역을 기반으로 정확히 EOD(일 마감) 총 자산과 수익률이 `account_daily_balance`에 기록(Upsert)되는지 검증.
* **TC-03: '오늘' 날짜 방어(Validation) 로직 검증**
  - 강제로 종료일을 '오늘' 혹은 '내일'로 세팅하고 API를 날렸을 때, `account_daily_balance`에 오늘(미래) 날짜 레코드가 생성되지 않고 철저히 어제 날짜까지만 반영되는지 확인.
* **TC-04: UI 실시간 PKI 표기 분리 확인**
  - 오늘 장중에 주식을 추가 매수하여 평가금액이 변동했을 때, 대시보드 당일 실시간 PKI가 즉시 변동하는 반면 `account_daily_balance` (어제자 데이터)는 오염되지 않고 그대로 유지되는지 테스트.
* **TC-05: 계좌개설일(dt_opened) 기반 하한 방어 검증**
  - 특정 계좌의 개설일 이전 날짜를 시작일(`start_date`)로 지정하여 요청했을 때, 시스템이 하한선을 `dt_opened`로 강제 조정하여 유효한 구간만 연산하는지 확인.
