## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. 보유 자산 상세 테이블 (DataTable) 연산 및 렌더링 스펙
백엔드(`GET /api/accounts`)가 제공하는 최신 전일 종가(`current_price` 필드)를 수신하여 메인 화면의 보유 자산 DataTable에 완벽하게 수식을 결합하라.

- **[현재가] 컬럼:** 백엔드가 동기화하여 넘겨준 전일자 종가 데이터를 천 단위 콤마 포맷(`toLocaleString()`)을 적용해 출력하라.
- **[평가금액] 컬럼:** `보유 수량(quantity) × 전일 종가(current_price)`
- **[수익률] 컬럼:** 
  - 공식: `((전일 종가 - 매입 평단가) / 매입 평단가) × 100`
  - 컬러 브레이크 규칙: 연산 결과가 플러스(+)이면 **Red 톤 텍스트**, 마이너스(-)이면 **Blue 톤 텍스트**로 마킹하여 금융 가시성을 확보하라. (소수점 아래 둘째 자리 고정 `.toFixed(2)`)

### 3. 화면 리프레시 프로토콜
- 매매 내역 등록/수정/삭제 모달이 닫히는 순간 자산 조회 API가 재트리거(`Refetch`)되면서 백엔드 캐시 테이블의 최신 마감 종가와 보유 주식 변동분이 조화롭게 결합해 화면에 즉시 렌더링되는지 확인하라.
