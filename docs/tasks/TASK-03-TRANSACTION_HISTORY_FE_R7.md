## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. 검색 연동성 퀄리티 크로스 체크
- **UI 인터랙션 유지:** R6에서 완성한 `p-inputgroup` 및 `isSearching` 로딩 스핀 로직을 그대로 유지하라.
- **실전 벤치마킹 테스트:**
  1. 매매 등록 창을 열고, 한 번도 검색하지 않았던 새로운 종목(예: "한화오션" 등)을 입력 후 검색 버튼을 눌러 로딩 스핀이 도는 시간을 체크하라 (최초 외부 로드 단계).
  2. 모달을 닫지 않은 상태 혹은 재오픈 후, 동일한 단어인 "한화"를 입력하고 검색 버튼을 다시 눌러라.
  3. 백엔드의 DB 캐싱 시스템 가동으로 인해 **두 번째 검색부터는 로딩 스핀이 거의 인지되지 않을 정도로 리얼타임 초고속 매핑**이 완료되어 InputBox가 채워지는지 눈으로 최종 확인하라.
