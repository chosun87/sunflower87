## 🌍 [BE_TASK] 백엔드 상세 구현 지침

### 1. 작업 디렉토리: `be/`

### 2. SQLite 데이터베이스 스키마 확장 (Multi-Account 대응)
- `transactions` 테이블 및 `stocks` 테이블에 **`account_number` (String)** 컬럼을 추가하라.
- 이로 인해 `stocks(현재고)` 테이블 내의 종목 수량과 평단가는 복합 키(Composite Key: `account_number` + `code`) 구조를 기준으로 독립 계산되도록 쿼리를 전면 패치하라.

### 3. CRUD API 엔드포인트 및 자산 역산(Rollback) 로직 구현
- **`POST /api/transactions/add`:** 
  - Pydantic DTO에 `account_number`를 필수로 수신하고, 지정된 계좌의 주식 잔고만 정밀 타겟팅하여 수량과 평단가를 `UPSERT` 하라. (미입력 시 현재시각 주입 로직 유지)
- **`DELETE /api/transactions/{id}` (★핵심):**
  - 삭제 대상 매매 기록을 먼저 조회한 뒤, 해당 계좌(`account_number`)의 현재고 잔고를 추적하여 **역산 연산**을 수행하라. (매수 삭제 시 잔고 차감 및 이전 평단가 복원, 매도 삭제 시 잔고 합산)
  - 역산 결과 잔고 수량이 0이 되면 `stocks` 테이블에서 해당 행을 `DELETE` 하고, 마이너스(-) 과매도 상태가 될 경우 트랜잭션을 롤백하고 `400 Bad Request` 에러를 반환하라.
- **`PUT /api/transactions/{id}`:**
  - [기존 거래 데이터 기반 자산 롤백] -> [거래 데이터 수정] -> [새로운 수정 데이터 기반 자산 재반영] 파이프라인을 단일 DB 트랜잭션(`db.commit()`) 내에서 수행하라.
