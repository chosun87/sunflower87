## 📌 [COMMON] 공통 요구사항
- **DB 데이터 기반 UI 렌더링:** 신규 매매 등록 화면의 주식계좌 선택 영역에 백엔드가 DB에서 받아온 최신 `account` 테이블 데이터를 동적으로 바인딩한다.
- **필드명 매핑 보정:** 백엔드 API 규격 개정에 따라 거래 요청 바디의 계좌 식별자 키값을 **`acc_code`**로 명확히 지정하여 전송한다.

## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. 주식계좌 선택 Dropdown 컴포넌트 고도화
- `@components/PrimeReact`에서 `Dropdown`을 임포트하여 매매 등록 Form 모달 내에 배치하라.
- **데이터 API 바인딩:**
  - 화면이 마운트될 때 백엔드의 `GET /api/accounts` API를 호출하여 활성화된 계좌 리스트를 Fetch 하라.
  - 불러온 데이터를 Dropdown의 `options` 속성에 바인딩하라.
- **Dropdown 속성 정의 (★백엔드 약어 컬럼 매핑):**
  - **`optionValue="acc_cd"`** : 백엔드로 전송할 계좌의 고유 코드 키값은 `acc_cd`를 매핑하라.
  - **`optionLabel="acc_nm"`** : 사용자 화면에 노출될 텍스트 필드는 계좌 별칭인 `acc_nm`으로 지정하라.
  - `placeholder="거래가 발생한 계좌 선택"` 문구를 적용하라.

### 3. 거래 내역 리스트(DataTable) 및 액션 동기화
- 하단 거래 내역 `DataTable`의 [거래 계좌] 컬럼에 해당 거래의 `acc_nm` (또는 필요시 `acc_company_nm` 조합)이 출력되도록 셀 템플릿을 조정하라.
- 사용자가 선택한 계좌 데이터는 폼 제출 시 백엔드가 지정한 **`acc_code`** 파라미터명에 실어서 `POST /api/transactions/add`로 전송하라.
- 매매 기록 등록/수정/삭제 성공 시, 대시보드의 자산 현황 테이블과 거래 내역 테이블이 동시에 최신 상태로 새로고침(`Refetch`) 되도록 상위 State를 체닝하라.
