## 📌 [COMMON] 공통 요구사항
- **그리드 0원 가시성 통제 (★최종 확정):** 백엔드에서 총 매수금액(`total_purchase_amt`)이 `0`으로 리턴될 경우, 화면 그리드 상에서 불필요한 컬러 브레이크(파란색/빨간색)가 돌지 않도록 무색(기본 텍스트 색상)으로 렌더링을 제어한다.
- **가시성 통제 규칙:** 수량과 금액은 우측 정렬(`align-right`), 숫자형 컬럼은 고정폭 폰트 클래스(`.monospace`)를 엄격하게 바인딩한다.

## 🌍 [FE_TASK] 프런트엔드 상세 구현 지침

### 1. 작업 디렉토리: `fe/`

### 2. [탭 1: 보유 자산 상세] DataTable Zero-Value 컬러 브레이크 예외 처리
총 매수금액이 0원일 때의 화면 뷰 깨짐을 방지하기 위해 템플릿 컬러 판정 식을 아래와 같이 정밀 정정하라.

* **총 평가손익 및 수익률 컬럼 스타일 가이드:**
    - **[조건 1] `total_purchase_amt === 0` 인 경우:**
      - 글자 색상에 파란색/빨간색 테마를 주지 말고, **시스템 기본 텍스트 컬러**를 유지하라.
      - 텍스트 뒤에 붙는 기호(`+`, `-`)를 전면 숨김 처리하고 `0원` / `0.00%`로 깔끔하게 출력하라.
    - **[조건 2] 수치가 0보다 큰 경우 (`> 0`):** `color: var(--blue-600)` 적용
    - **[조건 3] 수치가 0보다 작은 경우 (`< 0`):** `color: var(--red-600)` 적용

### 3. [탭 2: 매매 내역 히스토리] DataTable 및 수동 등록 폼 (R5 규격 승계)
- 날짜, 시간, 수량, 단가, 금액, 세금+수수료(`tax_fee`) 컬럼에 `.monospace` 및 우측 정렬 속성을 일관되게 부여하라.
- 매수(BUY)는 파란색 배지, 매도(SELL)는 빨간색 배지로 마킹하라.
